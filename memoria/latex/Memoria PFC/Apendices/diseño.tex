\subsection{Entidades del sistema}

A continuación vamos a describir las entidades del sistema:


\subsubsection{User, usuario}

Un usuario es una persona que se registra en la plataforma para poder
hacer uso de ella, ya sea creando encuestas o contestándolas. Además
guarda los permisos que posee dicho usuario dentro de la plataforma,
pudiendo tener un rol de investigador o de voluntario.


\subsubsection{Survey, encuesta}

\emph{Survey} sería la cabecera de la encuesta, en la que están definidas
los atributos y las limitaciones de está, como puede ser la fecha
durante la que estará activa la encuesta, la duración de esta, o simplemente
el título y la descripción d eesta.


\subsubsection{Consent, consentimiento}

\emph{Consent} almacena los consentimientos que debe aceptar un usuario
para realizar una encuesta. Cada consentimiento pertenece a una encuesta.


\subsubsection{Section, sección}

\emph{Section} sirven para definir los distintas secciones organizativas
que posee una encuesta, a su vez una sección puede contener mas secciones.
Además en cada sección se puede definir si son secciones exclusivas
respecto a las secciones del mismo nivel, así como definir el nombre
y la probabilidad de realizar dicha sección de la encuesta. Al final
las secciones generan un árbol que cuelga de una encuesta.


\subsubsection{Question, pregunta}

\emph{Question} almacena una pregunta de la encuesta. Las preguntas
pueden ser de distinto tipo, cada una tiene su propia clase que hereda
de \emph{Question}. Cada pregunta está definida dentro de una sección.


\paragraph{QuestionText, preguntas de texto}

Almacena preguntas cuya respuesta es texto, almacena también las limitaciones
que pueda tener la respuesta, como puede ser si es un número entero,
decimal o viene limitado por una expresión regular.


\paragraph{QuestionYN, preguntas de si o no}

Hace referencia a las preguntas cuya respuesta es si o no.


\paragraph{QuestionChoice, preguntas de elección}

Contiene las distintas opciones posibles en las preguntas cuyo rango
de respuestas está predeterminado. También guarda información de como
representar estás opciones al usuario.


\paragraph{QuestionLikerScala, preguntas de escala likert}

Almacena la información necesaria para las preguntas likert.


\subsubsection{Condition, condición}

\emph{Condition} hace referencia a las preguntas dependientes de la
respuesta de otra pregunta. Guardando que pregunta depende de quién
así como las condiciones necesarias para mostrar dicha pregunta.


\subsubsection{Answer, respuesta.}

Guarda la respuesta que ha dado un usuario a una pregunta, así como
el tiempo empleada en ella.


\subsubsection{StateSurvey, estado de la encuesta}

Almacena la información relativa del estado en el que se encuentra
una encuesta por parte de un usuario, guardando entre otras cosas
la sección por la que va, el tiempo empleado o el estado de la encuesta.

Como se puede ver en la figura \ref{modulos_sistema}, se ha dividido

\begin{figure}
\includegraphics{\string"/home/julio/flask/frame_game_theory/memoria/latex/Memoria PFC/figures/módulos\string".png}

\protect\caption{Módulos del sistema}

Los módulos con lo que se puede interactuar visualmente a través de
un navegador tienen la siguiente estructura:
\begin{itemize}
\item \textbf{Vistas}: La vista de una función es donde una petición HTTP
cumple con la lógica del programa. Es decir cada función tiene definida
una ruta web por la que se puede acceder. En un modelo MVC, sería
parte del controlador. Es esta función es donde los datos de la petición
HTTP se traducen a una llamada a la lógica de la aplicación produciendo
una respuesta. 
\item \textbf{Formularios}: Es el lugar donde están definidas las clases
que contienen los formularios de un módulo con los que puede interactuar
un usuario, así como las distintas validaciones que sufren los formularios. 
\item \textbf{Plantillas}: Cuando la vista genera una respuesta, es la plantilla
quien se encarga de generar el código HTML y JavaScript, para representar
los resultados al usuario.
\end{itemize}

\subsection{Módulo de decoradores}

Este módulo contiene los decoradores usados para proteger las vistas
de los distintos módulos que se pueden acceder vía petición HTML,
así se evita que cualquier usuario pueda ejecutar una vista a la que
no tiene acceso. Esto se ha realizado mediante decoradores, ya que
permiten alternar de manera dinámica la funcionalidad de una función,
método o clase, sin tener que realizar ningún cambio de código en
la clase decorada.

Los decoradores disponibles son los siguientes:
\begin{itemize}
\item \textbf{login\_required}: Comprueba que el usuario ha ingresado en
la plataforma. 
\item \textbf{researcher\_requerid}: Comprueba que el usuario es un investigador. 
\item \textbf{finished\_survey}: Comprueba que el usuario ha finalizado
la encuesta. 
\item \textbf{belong\_researcher}: Valida que la encuesta, sección o pregunta
que que quiere modificar pertenece al investigador en cuestión y no
a otro. 
\item \textbf{valid\_survey}: Comprueba que la encuesta ya se ha publicado. 
\item \textbf{there\_is\_stateSurvey}: Valida que existe y es valido el
estado de la encuesta por parte de un usuario que quiere rellenar
una encuesta.
\end{itemize}
Por ejemplo en la figura \ref{fig:proteccion_vista} , se puede comprobar
que para acceder a la vista \emph{new\_survey}, que sirve para la
creación de una nueva encuesta, debemos acceder a la dirección \emph{http://servidor/survey/new},
pero antes de poder acceder, se comprueba que el usuario se haya autentificado
en el sistema, mediante el decorador \emph{@login\_required}, y que
además el usuario tenga permisos de investigador, \emph{@researcher\_required}.
De esta forma se puede proteger las páginas a los que los usuarios
pueden acceder.

\begin{figure}[!tbh]
 	\begin{center}
\begin{listing}[style=consola, numbers=none]
@blueprint.route('/survey/new', methods = ['GET', 'POST'])
@login_required
@researcher_required
def new_survey():
\end{listing}
		 \caption{Protección de una vista}
 		\label{fig:proteccion_vista}
	 \end{center}
\end{figure}

\subsection{Módulo de Autentificación}

En el módulo de autetificación permite las siguientes opciones:
\begin{itemize}
\item Registrarnos en la plataforma: Uno se puede registrar mediante el
uso de un correo.


A esta función se accede mediante la vista: \emph{\textquotedbl{}auth/register\textquotedbl{}.}

\item Validarnos mediante el uso de un correo y contraseña.


A esta función se accede mediante la vista: \emph{\textquotedbl{}auth/loginEmail\textquotedbl{}.}

\item Validarnos mediante el uso de un servidor OpenID.


A esta función se accede mediante la vista:\emph{ \textquotedbl{}auth/\textquotedbl{}
}o \emph{\textquotedbl{}auth/login\textquotedbl{}.}

\item Cerrar la sesión actual.


A esta función se accede mediante la vista: \emph{\textquotedbl{}auth/logout\textquotedbl{}.}

\end{itemize}

\subsection{Módulo del investigador}

Este módulo permite a los usuarios que sean investigadores, las siguientes
acciones:
\begin{itemize}
\item Listado de encuestas creadas por un investigador.

\begin{itemize}
\item A esta función se accede mediante las vistas: \emph{\textquotedbl{}researcher/survey/\textquotedbl{}}
o \emph{\textquotedbl{}researcher/survey/index\textquotedbl{}}.
\end{itemize}
\item Creación de encuestas: Durante la creación de una encuesta se comprueba
que todos los campos del formulario son validos, además permite definir
las siguientes opciones:

\begin{itemize}
\item Título.
\item Descripción: Se hace uso de la sintaxis Markdown para el enriquecimiento
del texto.
\item Número máximo de usuarios que pueden acceder a la encuesta, si es
que lo hay.
\item Fecha de inicio de la encuesta.
\item Fecha de finalización de la encuesta.
\item Tiempo máximo para la realización de la encuesta, si es que lo hay.
\end{itemize}

A esta función se accede mediante la vista: \textquotedbl{}researcher/survey/new\textquotedbl{}

\item Modificación de una encuesta: Se permite modificar cualquier campo
de una encuesta creada.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta a modificar.

\item Eliminación de una encuesta: Se permite eliminar una encuesta creada,
siempre que no haya sido publicada.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/deleteSurvey/<int:id\_survey>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta a eliminar.

\item Creación de consentimientos: Mediante un formulario, el investigador
informa de los consentimientos que debe aceptar el usuario para poder
participar en la encuesta. Hace uso de la sintaxis Markdown para el
enriquecimiento del texto.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/consent/add\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta.

\item Modificación de un consentimiento.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/consent/<int:id\_consent>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta, e id\_consent
el identificador del consentimiento

\item Eliminación de un consentimiento.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/consent/<int:id\_consent>/delete\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta, e \emph{id\_consent}
el identificador del consentimiento

\item Creación de secciones y subsecciones: El investigador puede definir
con total libertad el número de secciones y subsecciones que tiene
una encuesta, además permite definir las siguientes opciones:

\begin{itemize}
\item Título de la sección.
\item Descripción: Se hace uso de la sintaxis Markdown para el enriquecimiento
del texto.
\item Secuencia de la sección.
\item Porcentaje de número de usuarios que realizan la sección.
\end{itemize}

El usuario que realice la encuesta recorrerá en profundidad el árbol
generado por la encuesta. Si dos o mas ramas tienen la misma secuencia,
se elige al azar que rama recorrer primero. Si dos o mas ramas con
la misma secuencia, tienen un porcentaje menor que 1, este porcentaje
indica la probabilidad que hay para recorrer esa rama y no cualquiera
de las otras que tienen la misma secuencia.


A la función de añadir una sección a una encuesta dada, se accede
mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/section/new\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta.


A la función de añadir una sección a una sección dada, se accede mediante
la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/section/<int:id\_section>/new\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta, e \emph{id\_sectión}
el identificador de la sección de la que va a colgar la nueva sección.

\item Modificación de una sección o subsección.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/section/<int:id\_section>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta, e \emph{id\_sectión}
el identificador de la sección a modificar

\item Eliminación de una sección o subsección:


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/deleteSection/<int:id\_section>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta, e \emph{id\_sectión}
el identificador de la sección a eliminar.

\item Creación de preguntas: Dada una sección, el investigador puede añadir
las preguntas que desee a la sección. Las posibilidades que tiene
son las siguientes:

\begin{itemize}
\item Preguntas cuya respuesta es Si o No, las opciones disponibles son
las siguientes:

\begin{itemize}
\item Texto de la pregunta: Se hace uso de la sintaxis Markdown para el
enriquecimiento del texto.
\item Indicar si es una pregunta cuya respuesta es obligatoria o no.
\item Respuesta correcta a la pregunta si es que la tiene, sirve para crear
preguntas de control.
\item Número máximo de intentos que tiene el usuario para responder bien
a la pregunta.
\end{itemize}
\item Preguntas cuya respuesta es un texto, las opciones disponibles son
las siguientes:

\begin{itemize}
\item Texto de la pregunta: Se hace uso de la sintaxis Markdown para el
enriquecimiento del texto.
\item Indicar si es una pregunta cuya respuesta es obligatoria o no.
\item Respuesta correcta a la pregunta si es que la tiene, sirve para crear
preguntas de control
\item Número máximo de intentos que tiene el usuario para responder bien
a la pregunta.
\item Validar que el texto introducido por el usuario en la respuesta es:

\begin{itemize}
\item Número entero.
\item Número decimal.
\item Comprobarlo mediante una expresión regular definido en sintaxis Python.
\end{itemize}
\item Mostrar un mensaje de error personalizado si ha introducido en un
formato incorrecto la respuesta.
\end{itemize}
\item Preguntas cuya respuesta es una opción dada, las opciones disponibles
son las siguientes.

\begin{itemize}
\item Texto de la pregunta: Se hace uso de la sintaxis Markdown para el
enriquecimiento del texto.
\item Indicar si es una pregunta cuya respuesta es obligatoria o no.
\item Respuesta correcta a la pregunta si es que la tiene, sirve para crear
preguntas de control.
\item Número máximo de intentos que tiene el usuario para responder bien
a la pregunta.
\item Formato en el que se mostrará la pregunta, hay tres posibilidades:

\begin{itemize}
\item Horizontal: Se mostrará las opciones mediante un radio botón, en una
disposición horizontal.
\item Vertical: Se mostrará las opciones mediante un radio botón, en una
disposición vertical.
\item Select: Se mostrará las opciones mediante un formulario de selección.
\end{itemize}
\item Definición de las respuestas disponibles mediante la definición de
un rango de números, así como el salto entre ellos.
\item Definición de las respuestas disponibles añadiéndolas de una en una.
\end{itemize}
\item Preguntas cuya respuesta es una escala likert, las opciones disponibles
son las siguientes.

\begin{itemize}
\item Texto de la pregunta: Se hace uso de la sintaxis Markdown para el
enriquecimiento del texto.
\item Indicar si es una pregunta cuya respuesta es obligatoria o no.
\item Indicar el rango de la escala likert, que estará definida entre 0/1
y un número mayor que 1.
\item Etiqueta opcional para indicar el significado mínimo de la escala.
\item Etiqueta opcional para indicar el significado máximo de la escala
\end{itemize}

Todas las preguntas descritas anteriormente se pueden mostrar o no
dependiendo de la respuesta dada a una pregunta de la misma sección.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/section/<int:id\_section>/addQuestion\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta e \emph{id\_sectión}
el identificador de la sección donde se encuentra la pregunta.

\end{itemize}
\item Modificación de una pregunta:


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/section/<int:id\_section>/question/<int:id\_question>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta e \emph{id\_sectión}
el identificador de la sección donde se encuentra la pregunta, e \emph{id\_question}
el identificador de la pregunta a modificar

\item Eliminación de una pregunta:


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/Section/<int:id\_section>/deleteQuestion/<int:id\_question>\textquotedbl{}},
siendo \emph{id\_survey} el identificador de la encuesta e \emph{id\_section}
el identificador de la sección donde se encuentra la pregunta, e \emph{id\_question}
el identificador de la pregunta a eliminar

\item Duplicar sección: Permite duplicar una sección así como todas las
secciones y preguntas que cuelgan de su rama.


A esta función se accede mediante la vista:
\begin{itemize}
\item \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/duplicateSection/<int:id\_section>/section/<int:id\_section2>'},
siendo \emph{id\_survey} el identificador de la encuesta, \emph{id\_sectión}
el identificador de la sección a copiar, \emph{id\_section2} el identificador
de la sección donde se copiará.
\item \emph{\textquotedbl{}researcher/survey/<int:id\_survey>/duplicateSection/<int:id\_section>/survey/'},
siendo \emph{id\_survey} el identificador de la encuesta, \emph{id\_sectión}
el identificador de la sección a copiar, se copiará directamente sobre
la raiz, la cual es la encuesta.
\end{itemize}
\item Exportar una encuesta: Permite exportar a un fichero XML, la encuesta
seleccionada.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/exportSurvey/<int:id\_survey>\textquotedbl{}},
siendo \emph{id\_survey} la encuesta a exportar

\item Importar una encuesta: Permite importar una encuesta mediante un fichero
XML.


A esta función se accede mediante la vista:\emph{ \textquotedbl{}researcher/survey/new\textquotedbl{}}

\item Exportar estadísticas de una encuesta: Permite exportar en un fichero
CSV, los resultados de la encuesta seleccionada.


A esta función se accede mediante la vista: \emph{\textquotedbl{}researcher/survey/exportStats/<int:id\_survey>\textquotedbl{}},
siendo \emph{id\_survey} la encuesta a exportar

\end{itemize}
Las vistas anteriores solo están disponibles para investigadores,
además no se pueden editar encuestas de otros investigadores, ya que
todas las vistas están protegidas mediante el decorador\emph{ \textquotedbl{}@belong\_researcher\textquotedbl{}.}


\subsection{Módulo del encuestado}

Este módulo permite a los usuarios que sean voluntarios realizar rellenar
encuestas. Las acciones disponibles son las siguientes:
\begin{itemize}
\item Listar todas las encuestas disponibles, así como el estado de esta,

A esta función se accede mediante la vista: \emph{\textquotedbl{}survey/index\textquotedbl{}}
o \emph{\textquotedbl{}survey/\textquotedbl{}}

\item Empezar o continuar una encuesta: Es la función encargada de decidir
cual es el siguiente paso cuando se realiza una encuesta, ya sea mostrar

\item Mostrar los consentimientos: Esta función es la encargada de mostrar

siendo \emph{id\_survey} la encuesta a realizar 

\item Mostrar una sección: Esta función es la encargada de mostrar una sección,

A esta función se accede mediante las vistas \emph{\textquotedbl{}survey/<int:id\_survey>/section/<int:id\_section>\textquotedbl{}}

Todas las vistas están protegidas, de manera que es obligatorio haber

\subsection{Módulo de Feedback}

Este módulo permite dar feedback sobre las decisiones tomadas en el
proyecto \emph{\textquotedbl{}¿Cómo son nuestros voluntarios?\textquotedbl{}}.
Cada vista corresponde con una decisión concreta del experimento. 
\begin{itemize}
\item Lógica: Durante el desarrollo del proyecto \textquotedbl{}¿Cómo son
nuestros voluntarios?\textquotedbl{}, se decidió que el orden del
feedback sobre las decisiones tomadas, iba a ser el mismo que el realizado
durante el experimento, y debido a que cada voluntario se le asigna
un orden al azar, está función es la encargada de decidir que feedback
se le va a mostrar al voluntario, siempre siguiendo el mismo orden
de la encuesta.


A esta función se accede mediante la vista \textquotedbl{}feedback/id\_survey\textquotedbl{},
siendo id\_survey la encuesta de la que recibir feedback.

\item Juegos de la lotería, la renta y dictador: Corresponde a las decisiones
de las versiones de los dos juegos de la lotería y los dos juegos
de la renta. Muestra la respuesta del usuario y la media dada por
los demás voluntarios, además muestra el porcentaje de los voluntarios
que han tomado la misma decisión que el usuario.
\item Juego del ultimatum: A parte de mostrar la información que se da en
los demás juegos, también se muestra el porcentaje de voluntarios
que hubiesen aceptado la cantidad dada por el usuario.
\end{itemize}

\subsection{Módulo de estadísticas:}

encuestas genéricas y otra para el experimento \emph{\textquotedbl{}¿Cómo
son nuestros voluntarios?\textquotedbl{}.} Ya que se adapto los resultados
a la necesidad de los investigadores en cuanto al formato requerido
por ellos para trabajar más cómodamente.


\subsection{Módulo Juegos:}

Este módulo es el encargado de decidir en los juegos del experimento
\textquotedbl{}¿Cómo son nuestros voluntarios?\textquotedbl{}, que
usuarios se enfrentan con quién. Además de buscar las respuestas dadas
por los voluntarios.

A la hora de decidir que usuarios van a enfrentarse, se busca a todos
los usuarios que no han realizado ningún enfrentamiento. Si este llega
a un número mínimo necesario para que no existan enfrentamientos repetidos,
se continua. Sino se selecciona las decisiones tomadas de otros voluntarios
que ya se han enfrentado, pero sin posibilidad de que estos últimos
vuelvan a salir premiados.

\subsection{Módulo de configuración}

Este módulo se encarga de leer la configuración de la plataforma una
vez que se inicia está. Aunque no se ha señalado explícitamente, los
demás módulos interaccionan con él si tienen que leer alguna configuración
de la plataforma.

Se han tenido en cuenta las recomendaciones dadas en http://flask.pocoo.org/docs/config/\#configuration-best-practices

Y se ha decidido crear un sistema de clases y herencia para la configuración
del sistema, pudiendo cambiar el modo de funcionamiento de la aplicación
simplemente cambiando la variable de entorno \emph{FLASK\_CONFIG},
la estructura que sigue y los modos disponibles son los siguientes:

\begin{listing}[style=consola, numbers=none]
class Config(object):
    # configuración básica de la plataforma

class DevelopmentConfig(Config):
    # Configuración de la plataforma en modo desarrollo
    DEBUG = True

class TestingConfig(Config):
    # Configuración de la plataforma para test unitarios

class Jmeter(Config):
    # Configuración de la plataforma para la realización de pruebas de sobrecarga

class ProductionConfig(Config):
    # Configuración de la plataforma en modo producción
    # Se define quien es el usuario administrador y cual es el servidor de correo para enviar los distintos logs

class HerokuConfig(ProductionConfig):
    # Configuración de la plataforma en modo producción en la nube Heroku
    # Usa el sistema de logs de Heroku para almacenarlos.


class UnixConfig(ProductionConfig):
    # Configuración de la plataforma en modo producción en una máquina de tipo Unix
    # Hace uso del sistema de logs de la máquina Unix.
\end{listing}

Por otra parte, para facilitar el tener varias configuraciones posibles,
se carga primeramente la configuración básica definida anteriormente
y luego se puede reemplaza por la leída en la variable de entorno
\emph{SWARMS\_SURVEY\_SETTINGS}, pudiendo cambiar fácilmente la configuración.

\section{Prototipado de las ventanas y cuestiones de usabilidad}

Después del análisis y el diseño de las distintas partes de la aplicación,
se continuo con un diseño preliminar de la interfaz para el usuario.
Se tomaron distintas ideas de las plataformas estudiadas para la elaboración
de encuestas, así como las guías de usabilidad del proyecto KDE, http://techbase.kde.org/Projects/Usability/HIG

Para facilitar al investigador la navegación entre las distintas secciones,
se decidió mostrar el árbol de secciones por completo en un lateral
de la ventana. Además de mostrar en la parte superior un menú de tipo
\textquotedbl{}migas de pan\textquotedbl{}, que presenta en forma
textual todos los enlaces que describen la ruta de una sección a partir
de la raíz, siendo esta la propia encuesta.

Por otra parte para mejorar la usabilidad de la aplicación, se ha
añadido código JavaScript en distintas páginas para ofrecer ciertas
funcionalidades al usuario, sin tener que realizar peticiones a la
plataforma web, añadiendo velocidad y fluidez a la plataforma.

Se decidió también añadir un editor y visualizador de código Markdown
para poder comprobar en tiempo real el formato de texto que verían
los usuarios a la hora de realizar encuestas.

Por otra parte para mejorar la usabilidad de la plataforma, se uso
un sistema de URL amigables, para recordar mas fácilmente la funcionalidad
de cada página web.

Además durante el prototipado de las ventanas, se tuvo en cuenta las
funcionalidades que provee el framework Bootstrap, para facilitar
que las distintas páginas se adaptasen al tamaño del navegador.
